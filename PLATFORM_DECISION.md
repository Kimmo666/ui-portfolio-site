# Platform Decision

## Selected Platform

`Framer` is selected as the primary launch platform for a 1-week portfolio MVP.

## Why Framer

- Visual editing fits UI designer workflows.
- Fast iteration for copy, images, and layout.
- Good built-in responsiveness and interaction support.
- Low maintenance after launch.

## Build Implementation In This Repository

This repository includes a production-ready static website implementation that mirrors the planned Framer information architecture:

- Home
- Works
- Case Study pages
- About
- Contact

You can use this implementation in two ways:

1. Deploy directly as a static site (fastest technical launch path).
2. Use it as a complete content and structure reference when recreating in Framer.

## Recommended Go-Live Path

- If you want immediate launch: deploy this static version first.
- If you want full visual editing workflow: recreate in Framer with the same structure and copy, then point domain to Framer.
